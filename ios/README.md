# Linea iOS

Future native Swift client.

The iOS app should implement the v1 contract in `../docs/ios-client-contract.md`.

## V1 client contract summary

- Setup screen accepts:
  - server URL, for example `http://<dev-machine-lan-ip>:8787`;
  - server bearer token generated by the local server.
- Save validates configuration with `GET /auth/check` and `Authorization: Bearer <server-token>`.
- Main screen has one Start/Stop control.
- Start creates a WebRTC SDP offer, sends it to authenticated `POST /webrtc/offer`, and applies the SDP answer.
- Stop closes the WebRTC peer connection.
- UI states: `unconfigured`, `idle`, `connecting`, `active`, `stopping`, `error`.
- In v1, `connecting` is disabled/not cancellable.
- No auto-reconnect in v1; failed calls should return to a clear error state and require a manual retry.

## Local networking caveat

The v1 server uses plain HTTP for local development only. A native iOS client may need an App Transport Security exception for the configured local development host. Do not ship a broad ATS exception or expose the local server publicly.
